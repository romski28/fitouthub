# "Next Step" Button Feature Specification

## Overview
Contextual action buttons that guide users through the project lifecycle, appearing at key decision points. Reduces cognitive load by highlighting the most relevant action at each stage.

---

## Project Stage Lifecycle & Next Steps

### 1. **PROJECT CREATED** 
*Client has just submitted project for bidding*

**Client Next Steps:**
- "Wait for quotes" (primary) → Shows expected timeline
- "Invite professionals" (proactive) → Add specific professionals to invitations
- "View project details" (reference) → Review what you submitted
- "Share project" (optional) → Email/link to external professionals

**Professional Next Steps:** (only if invited)
- "Reply to invitation" (primary) → View details and submit quote
- "Decline" (optional) → Remove invitation

---

### 2. **BIDDING PHASE - ACTIVE**
*Professionals are submitting/can submit quotes. Bidding window open.*

**Client Next Steps:**
- "Review incoming quotes" (primary) → See how many quotes received
- "Send reminder" (optional) → Follow up with non-responding professionals
- "Close bidding early" (optional) → If enough quotes received and ready to choose
- "Request site visit" (elective) → Schedule visits from interested professionals
- "Edit project" (reference) → Update details/budget/scope

**Professional Next Steps:**
- "Submit quote" (primary if not yet submitted) → Provide pricing/timeline
- "Update quote" (primary if already submitted) → Modify existing bid
- "Request site access" (elective) → Schedule time to visit property
- "Decline" (optional) → Withdraw from bidding

---

### 3. **BIDDING PHASE - QUOTE RECEIVED**
*Professional has submitted quote, waiting for client decision*

**Professional Next Steps:**
- "Request site access" (elective) → If not done, schedule visit for measurements
- "Wait for client decision" (status) → Shows estimated decision date
- "Update quote" (optional) → Modify your bid if needed
- "View competitors' quotes" (reference) → See if you're in range (if client allows sharing)

---

### 4. **BIDDING PHASE - SITE VISIT REQUESTED/SCHEDULED**
*Professional has requested or scheduled site access*

**Client Next Steps:**
- "Schedule site visit" (primary) → Confirm date/time for professional
- "Provide access details" (primary) → Add gate codes, parking info, contact on-site
- "Cancel site visit" (optional) → Remove this appointment
- "Review quotes while waiting" (reference) → Continue comparing bids

**Professional Next Steps:**
- "Confirm visit time" (primary) → Accept/modify proposed date
- "View access details" (reference) → Get keys/codes/contact info from client
- "Cancel visit" (optional) → Reschedule or withdraw
- "Prepare site visit notes" (optional) → Add checklist of what to measure/inspect

---

### 5. **SITE VISITS COMPLETED**
*Professional(s) have visited site and inspected/measured*

**Professional Next Steps:**
- "Submit revised quote" (primary) → Provide detailed estimate based on site visit
- "Complete inspection report" (elective) → Document findings, recommendations
- "Update quote" (if already submitted) → Modify based on findings

**Client Next Steps:**
- "Review updated quotes" (primary) → Compare revised bids from professionals
- "Request clarifications" (optional) → Ask questions about quotes/scope
- "Continue waiting" (status) → Show which professionals haven't submitted revised quotes

---

### 6. **BIDDING CLOSED - REVIEWING QUOTES**
*All bidding window closed or client ready to choose. Multiple quotes available.*

**Client Next Steps:**
- "Select professional" (primary) → Choose winner and accept quote
- "Request price revision" (optional) → Negotiate with top candidates
- "Reject quote & reopen bidding" (optional) → Send project back out for new bids
- "Compare quotes side-by-side" (reference) → View detailed comparison
- "Contact professional" (optional) → Message for clarification before deciding

**Professional (selected):**
- "Prepare contract" (primary) → Move to contract negotiation phase
- "Schedule kickoff meeting" (elective) → Set meeting with client
- "Confirm milestones & timeline" (primary) → Agree on project schedule

**Professional (not selected):**
- "Request feedback" (optional) → Ask why you weren't chosen
- "Reapply if project reopens" (status) → Bid again if client rejects all quotes
- "View decision" (status) → See which professional was selected

---

### 7. **PROFESSIONAL SELECTED - CONTRACT PHASE**
*One professional chosen. Now negotiating terms.*

**Client Next Steps:**
- "Review contract" (primary) → Check terms, payment schedule, warranty
- "Approve contract" (primary) → Accept and move to work phase
- "Negotiate terms" (optional) → Discuss payment schedule/warranty/insurance
- "Schedule kickoff meeting" (elective) → Plan project start
- "Request changes" (optional) → Modify scope/timeline before signing

**Professional Next Steps:**
- "Submit contract" (primary) → Present contract terms and payment schedule
- "Update contract" (if requested) → Modify terms based on client feedback
- "Confirm milestones" (primary) → Agree on payment schedule & completion dates
- "Schedule kickoff meeting" (elective) → Plan work start date
- "Provide insurance/license docs" (optional) → Upload credentials for client review

---

### 8. **CONTRACT SIGNED - PRE-WORK**
*Both parties agreed. Project about to start.*

**Client Next Steps:**
- "Schedule project kickoff" (primary) → Set start date and first meeting
- "Prepare property" (elective) → Clear site, arrange access, utilities
- "Arrange payment method" (primary) → Confirm how payments will be made (invoice, app, etc.)
- "Request payment schedule" (reference) → See milestone dates/amounts
- "Enable notifications" (optional) → Get alerts on progress/milestones

**Professional Next Steps:**
- "Confirm project start date" (primary) → Agree with client on kickoff
- "Schedule site orientation" (elective) → Plan first day walkthrough
- "Prepare work schedule" (elective) → Create detailed timeline
- "Check insurance/licensing" (reference) → Ensure everything is current
- "Request access arrangements" (elective) → Confirm keys/codes/parking

---

### 9. **PROJECT ACTIVE - WORK IN PROGRESS**
*Work has started. Ongoing execution phase.*

**Client Next Steps:**
- "Review progress photos" (primary) → Check latest updates
- "Approve upcoming milestone" (primary) → Confirm work ready for payment
- "Contact professional" (optional) → Ask questions or report issues
- "Request change order" (elective) → Modify scope mid-project
- "View project timeline" (reference) → See schedule vs. actual progress

**Professional Next Steps:**
- "Submit progress update" (primary) → Post photo/video of work completed
- "Request milestone payment" (primary) → When work phase complete, request payment
- "Report issue/delay" (elective) → Notify client of problems affecting timeline
- "Request change order approval" (optional) → If scope expansion needed
- "Schedule client walkthrough" (elective) → Invite inspection of current work

---

### 10. **MILESTONE PENDING APPROVAL**
*Professional completed phase of work, requesting payment*

**Client Next Steps:**
- "Inspect milestone work" (primary) → Review photos/schedule site visit
- "Approve & release payment" (primary) → Accept work, authorize payment
- "Request corrections" (optional) → List items needing rework before approval
- "Negotiate timeline" (optional) → If work is delayed
- "Schedule walkthrough" (elective) → Inspect in person before approving

**Professional Next Steps:**
- "Wait for approval" (status) → See client's review deadline
- "Provide documentation" (optional) → Submit invoices, certificates, receipt photos
- "Follow up" (optional) → Remind client if approval delayed
- "Prepare next phase" (proactive) → Start planning next milestone work

---

### 11. **PAYMENT APPROVED - PAYMENT RELEASED**
*Client approved milestone. Payment processed/pending.*

**Professional Next Steps:**
- "Confirm payment received" (primary) → Mark payment as processed
- "Proceed to next phase" (primary) → Start next stage of work
- "Provide payment receipt" (optional) → Send invoice/receipt to client
- "Update project timeline" (reference) → Adjust remaining schedule

**Client Next Steps:**
- "Confirm payment sent" (reference) → Verify transaction completed
- "Review next phase" (optional) → See what's coming
- "Enable ongoing notifications" (optional) → Get alerts for next milestone

---

### 12. **PROJECT NEARING COMPLETION**
*Final phase of work. Nearly done.*

**Professional Next Steps:**
- "Schedule final walkthrough" (primary) → Invite client to inspect completed work
- "Request final payment" (primary) → Submit invoice for final milestone
- "Provide project summary" (elective) → Document all work completed, warranties
- "Request testimonial/review" (elective) → Ask client for feedback

**Client Next Steps:**
- "Schedule final inspection" (primary) → Plan walkthrough with professional
- "Review punch list" (elective) → Note any minor touchups needed
- "Approve final payment" (primary) → Ready to pay for completed work

---

### 13. **PROJECT COMPLETE - FINAL INSPECTION**
*Work finished. Client reviewing final product.*

**Client Next Steps:**
- "Approve final work" (primary) → Mark project as complete
- "Release final payment" (primary) → Pay outstanding invoice
- "Request punch list items" (optional) → Ask for final touch-ups
- "Request warranty documentation" (elective) → Get warranty terms/coverage
- "Leave review/testimonial" (primary - encourages feedback) → Rate professional

**Professional Next Steps:**
- "Confirm final approval" (status) → See if client approved work
- "Wait for final payment" (status) → Check payment status
- "Send warranty details" (elective) → Provide warranty documentation
- "Request testimonial/review" (primary - encourages feedback) → Ask for rating

---

### 14. **PROJECT CLOSED - POST-COMPLETION**
*Work finished, paid, possibly reviewed*

**Client Next Steps:**
- "Leave review/testimonial" (if not done) → Rate professional
- "Save project documentation" (optional) → Download warranty/invoices/photos
- "Request warranty support" (elective) → Report issues within warranty period
- "Start new project" (proactive) → Begin another project with same or different professional

**Professional Next Steps:**
- "Collect testimonial" (if not done) → Follow up for review/rating
- "Request referral" (elective) → Ask for recommendations to others
- "Offer future services" (proactive) → Propose maintenance/warranty follow-ups
- "View project case study" (optional) → See before/after photos in your portfolio

---

### 15. **BIDDING REJECTED - NO WINNER SELECTED**
*Bidding closed but client rejected all quotes or reopened*

**Client Next Steps:**
- "Reopen project for new bids" (primary) → Send project back out with updated info
- "Increase budget" (optional) → Raise budget if quotes too high
- "Adjust scope" (optional) → Reduce scope if cost is issue
- "Contact prior bidders" (optional) → Ask if they'd resubmit at higher budget

**Professional (previously bidding):**
- "Reapply to project" (optional) → Submit new quote if project reopened
- "Request feedback" (optional) → Ask why your quote wasn't chosen

---

### 16. **PROJECT PAUSED/SUSPENDED**
*Work on hold. Client or professional initiated pause.*

**Client Next Steps:**
- "Resume project" (primary) → Restart work with agreed schedule
- "Cancel project" (optional) → Terminate if won't continue
- "Reschedule" (elective) → Set new work dates

**Professional Next Steps:**
- "Confirm resume date" (primary) → Agree on when work restarts
- "Update timeline" (elective) → Adjust project schedule
- "Confirm crew availability" (optional) → Verify team still available for new dates

---

### 17. **PROJECT DISPUTED/ISSUE**
*Disagreement on work quality, timeline, or payment*

**Client Next Steps:**
- "Send formal complaint" (primary) → Document issue for resolution
- "Request mediation" (optional) → Involve platform team
- "Propose resolution" (optional) → Suggest acceptable fix/compensation
- "Report professional" (optional) → Flag for quality issues

**Professional Next Steps:**
- "Respond to complaint" (primary) → Address client's concerns
- "Propose resolution" (optional) → Offer fix or compensation
- "Request mediation" (optional) → Involve platform team
- "Provide evidence" (optional) → Document work quality/timeline adherence

## UI Placement Strategy

### Primary Locations (in priority order):

1. **Project Dashboard/Overview Card**
   - Shows next step & current stage
   - One-click action button
   - Appears when user logs in (personalized per user/role)

2. **Project Detail Page**
   - Prominent "Next Step" button at top
   - Shows estimated next review/action date
   - Lists secondary optional actions

3. **Updates/Notifications Panel**
   - When new action required, show prioritized suggestion
   - Badge showing count of projects needing action

4. **My Projects List**
   - Small badge/label showing current status
   - Click to see primary next step without opening project

### Secondary Locations:

5. **Email/Push Notifications**
   - "Action required: [Next Step]" prompts
   - Link directly to action

6. **Mobile App**
   - Card-based UI showing primary next step
   - One-tap action buttons

7. **Project Timeline View**
   - Visual indicator of where project is
   - Next milestone action highlighted

---

## Implementation Considerations

- **Conditional Logic**: Show next step based on role (client vs. professional), project stage, and permissions
- **Smart Defaults**: Suggest most common action first, but allow selecting  alternative actions
- **Notifications**: Alert user when new next step becomes available
- **Accessibility**: Ensure guidance system helps, not hinders power users
- **Mobile-First**: Compact button label + tooltip on desktop, full text on mobile
- **Internationalization**: Text should be translatable for multilingual support
- **Undo/Go Back**: Allow users to not take suggested action without penalty
- **Analytics**: Track which next steps are most used/ignored for UX optimization

## Data Model Changes

```typescript
// Add to Project model:
interface Project {
  // ... existing fields ...
  
  // Stage tracking
  currentStage: ProjectStage; // Enum: CREATED, BIDDING, SITE_VISIT, BIDDING_CLOSED, CONTRACT, ACTIVE, MILESTONE_PENDING, COMPLETE, CLOSED, PAUSED, DISPUTED
  stageStartDate: Date;
  
  // Action tracking (optional, for analytics)
  lastNextStepSuggestedAt?: Date;
  lastNextStepAction?: string;
}

// New table for tracking user actions
interface ProjectAction {
  id: string;
  projectId: string;
  userId: string;
  actionType: string; // e.g., "SELECTED_PROFESSIONAL", "APPROVED_MILESTONE"
  stage: ProjectStage;
  timestamp: Date;
  metadata?: Record<string, any>;
}
```

## Example "Next Step" Text (UX Writing)

- **Clear & Action-Oriented**: "Review quotes" not "Quote status"
- **Role-Specific**: Different language for client vs. professional  
- **Timeline-Aware**: "Wait for quotes (3 more days)" not just "Wait"
- **Reassuring**: Guides users without pressure
- **Progressive Disclosure**: Show primary action, hide secondary options in dropdown
